หลายครั้งที่นักพัฒนาซอฟต์แวร์จำเป็นต้องรันโปรแกรมในสภาพแวดล้อมปิด (isolated environment) เพื่อให้โปรแกรมทำงานได้อย่างถูกต้องโดยปราศจากรบกวนจากปัจจัยภายนอก หรือเพื่อป้องกันไม่ให้โปรแกรมส่งผลกระทบต่อโปรแกรมอื่นๆ บนเครื่อง มักมีการใช้ Virtual Machine (VM) เป็นวิธีแก้ปัญหา

อย่างไรก็ตาม การใช้ VM นั้นมีข้อจำกัดอยู่หลายประการ ประการแรก เครื่องคอมพิวเตอร์ของผู้ใช้จำเป็นต้องใช้ทรัพยากรระบบจำนวนมากเพื่อจำลองและรันระบบปฏิบัติการ (OS) ไว้ภายใน VM หนึ่งตัว ทรัพยากรเหล่านี้รวมถึง CPU, RAM และ ROM ซึ่งเปรียบเสมือนการมีคอมพิวเตอร์อีกเครื่องมาแบ่งใช้ทรัพยากรร่วมกัน แต่เพียงเพื่อรันโปรแกรมเพียงตัวเดียวเท่านั้น ปัญหาจะยิ่งทวีความรุนแรงมากขึ้นเมื่อต้องการรันโปรแกรมหลายโปรแกรมพร้อมกัน ส่งผลต่อทั้งการใช้ทรัพยากร ความเร็วในการเปิดและปิดโปรแกรม และความยุ่งยากในการจัดการแก้ไข

ปัญหาทั้งหมดนี้สามารถหลีกเลี่ยงได้ด้วยการใช้ Docker แทน VM

Docker เป็นแพลตฟอร์มสำหรับการรันแอปพลิเคชันในลักษณะของคอนเทนเนอร์ (container) คอนเทนเนอร์เปรียบเสมือนกล่องบรรจุภัณฑ์ที่มัดรวมแอปพลิเคชัน ไลบรารี และไฟล์ที่จำเป็นทั้งหมดไว้ด้วยกัน ทำให้สามารถรันแอปพลิเคชันได้อย่างแยกตัวโดยไม่ต้องพึ่งพาระบบปฏิบัติการโฮสต์

### ข้อดีของ Docker เทียบกับ VM :

\- \*\*ทรัพยากรระบบน้อยลง:\*\* คอนเทนเนอร์ Docker ใช้ทรัพยากรระบบน้อยกว่า VM มาก เนื่องจากไม่จำเป็นต้องจำลองระบบปฏิบัติการแยกต่างหาก

\- \*\*รวดเร็ว:\*\* คอนเทนเนอร์ Docker สามารถเริ่มทำงานได้อย่างรวดเร็วกว่า VM มาก

\- \*\*ง่ายต่อการจัดการ:\*\* คอนเทนเนอร์ Docker นั้นง่ายต่อการจัดการและปรับใช้มากกว่า VM

\- \*\*แยกตัว:\*\* คอนเทนเนอร์ Docker แยกตัวจากกัน ทำให้สามารถรันแอปพลิเคชันหลายตัวพร้อมกันบนเครื่องเดียวโดยไม่ต้องกังวลว่าจะรบกวนกัน แยกตัวอิสระจาก โฮสต์ ใช้ทรัพยากรร่วมกับ เครื่องคอมพิวเตอร์ server ใช้ RAM , CPU หรือ storage Hardisk ร่วมกันได้

### ส่วนประกอบหลักของ Docker

Docker ประกอบไปด้วยสองส่วนประกอบหลัก ดังนี้

\*\*1. Docker Image:\*\*

\- เปรียบเสมือนแม่พิมพ์ หรือ เทมเพลต ที่ใช้สร้างคอนเทนเนอร์

\- ประกอบด้วยระบบปฏิบัติการ โปรแกรม ไลบรารี และไฟล์ที่จำเป็นต่อการรันแอปพลิเคชัน

\- ถูกเก็บไว้ใน Docker Registry รอการดึงใช้งาน

\- เปรียบเสมือนต้นแบบที่สามารถสร้างคอนเทนเนอร์หลายตัวขึ้นมาได้โดยใช้ Image ตัวเดียวกัน

\*\*2. Docker Container:\*\*

\- เปรียบเสมือนตู้คอนเทนเนอร์ที่บรรจุแอปพลิเคชันและรันแอปพลิเคชันนั้น

\- สร้างจาก Image

\- แยกตัวออกจากกัน ไม่ส่งผลกระทบต่อระบบปฏิบัติการโฮสต์ หรือคอนเทนเนอร์อื่นๆ

\- สามารถรันหลายคอนเทนเนอร์บนเครื่องเดียวพร้อมกัน

\- เปรียบเสมือนแอปพลิเคชันที่พร้อมใช้งาน เพียงแค่เปิดใช้งานก็สามารถใช้งานได้ทันที

### กล่าวโดยสรุปประเด็นสำคัญเกี่ยวกับ Docker

Docker เป็นแพลตฟอร์มที่ช่วยให้ผู้ใช้สามารถบรรจุแอปพลิเคชันและ Dependency ต่างๆ ลงในรูปแบบของคอนเทนเนอร์ (container) ทำให้มั่นใจได้ว่าแอปพลิเคชันจะสามารถทำงานได้อย่างถูกต้องทุกที่ที่ต้องการ โดยมีจุดเด่นดังนี้

\*\*1. ความง่าย (Simplicity):\*\*

\- แม้จะเป็นแอปพลิเคชันที่ซับซ้อน ก็สามารถบรรจุลงในคอนเทนเนอร์ Docker ได้อย่างง่ายดาย

\- ไม่จำเป็นต้องติดตั้งซอฟต์แวร์เพิ่มเติมบนเครื่องที่ต้องการรันแอปพลิเคชัน

\- ผู้ใช้สามารถโฟกัสไปที่การพัฒนาแอปพลิเคชันโดยไม่ต้องกังวลเรื่องสภาพแวดล้อม

\*\*2. ความรวดเร็ว (Agility):\*\*

\- การติดตั้งแอปพลิเคชันที่รวดเร็ว

\- สามารถเริ่มต้นใช้งานแอปพลิเคชันได้ทันทีโดยไม่ต้องรอนาน

\- ปรับขนาดแอปพลิเคชันได้ง่าย

\- นำแอปพลิเคชันไปรันบน Cloud ได้อย่างรวดเร็ว

\*\*3. ทางเลือกที่หลากหลาย (Choice):\*\*

\- รองรับภาษา เฟรมเวิร์ค และระบบปฏิบัติการที่หลากหลาย

\- ผู้ใช้สามารถเลือกใช้เครื่องมือที่เหมาะสมกับความต้องการ

\- สามารถนำแอปพลิเคชันไปติดตั้งบนผู้ให้บริการ Cloud ได้ทุกที่

### \*\*ตัวอย่างการใช้งาน Docker:\*\*

\- นักพัฒนาซอฟต์แวร์สามารถใช้ Docker เพื่อรันแอปพลิเคชันในสภาพแวดล้อมที่แยกตัว ช่วยให้มั่นใจได้ว่าแอปพลิเคชันจะทำงานได้ถูกต้องโดยไม่รบกวนระบบอื่นๆ

\- ทีม DevOps สามารถใช้ Docker เพื่อปรับใช้แอปพลิเคชันใหม่ได้อย่างรวดเร็วและมีประสิทธิภาพ

\- องค์กรสามารถใช้ Docker เพื่อรันแอปพลิเคชันบน Cloud โดยไม่ต้องกังวลเรื่องความเข้ากันได้ของระบบ

### เรามาลองใช้งาน docker กันเลย

สำหรับการใช้งาน Docker นั้นจะต้องใช้คำสั่งในการสั่งงาน ผ่าน Command Line

เมื่อติดตั้งเสร็จแล้ว ให้เปิดโปรแกรม Terminal หรือ cmd แล้วมาเริ่มกันที่การใช้คำสั่ง Docker กันเลยครับ

หลังจากติดตั้ง docker แล้ว ลองตรวจสอบเลข Version กันก่อนเลย ด้วยคำสั่งนี้

```dockerfile
docker --version
```

หรือ

```dockerfile
docker -v
```

### \*\*เริ่มจากการ Pull Docker Image\*\*

ก่อนจะทำการสร้างและรัน Docker container ลองค้นหาใน ==\[Docker Hub]\(https\://hub.docker.com/)==ก่อนครับว่ามีอะไรให้เราใช้บ้าง

!\[\[Screenshot 2567-05-13 at 10.50.30.png]]

เมื่อเจอ Docker image ที่ต้องการ สามารถโหลด Docker image ที่ต้องการด้วยคำสั่งด้านล่างนี้

```dockerfile
docker pull
```

ตัวอย่าง : ในที่นี้จะโหลด Docker image ที่ชื่อว่า nginx ให้ใช้คำสั่ง

```dockerfile
docker pull nginx
```

หรือ

```dockerfile
docker pull nginx:stable
```

เราสามารถเลือกเวอร์ชั่นของ image ได้ด้วยการระบุ tag ลงไป หากไม่ระบุ tag ก็จะเป็นการเลือกใช้เวอร์ชั่นล่าสุด เช่น nginx ก็จะมี Docker Image ให้เลือกหลายๆเวอร์ชั่น nginx\:stable, nginx:1.10

### การสร้าง Container

มาถึงขั้นตอนสำคัญ ก็คือการสร้าง Container โดยใช้คำสั่ง docker run เพื่อสร้าง Docker Container

### ตัวอย่าง

```dockerfile
docker run--name demo - d - v / home / me: /home/docker - p 8080: 80 image nginx
```

### \*\*คำสั่ง Docker run ที่ใช้ในตัวอย่างนั้นมี option ดังนี้:\*\*

\- \*\*--name demo:\*\* ตั้งชื่อคอนเทนเนอร์ว่า "demo" ช่วยให้สามารถเรียกใช้คอนเทนเนอร์ด้วยชื่อนี้แทนการใช้ ID ของคอนเทนเนอร์

\- \*\*-d:\*\* รันคอนเทนเนอร์ในโหมด background ช่วยให้คอนเทนเนอร์ทำงานต่อหลังจากผู้ใช้ปิด terminal

\- \*\*-v /home/me:/home/docker:\*\* แชร์โฟลเดอร์ "/home/me" บนเครื่องโฮสต์ไปยังโฟลเดอร์ "/home/docker" ภายในคอนเทนเนอร์ ช่วยให้สามารถเข้าถึงและแก้ไขไฟล์ในโฟลเดอร์เหล่านี้ได้จากทั้งบนเครื่องโฮสต์และภายในคอนเทนเนอร์

\- \*\*-p 8080:80:\*\* map พอร์ต 8080 บนเครื่องโฮสต์ไปยังพอร์ต 80 ภายในคอนเทนเนอร์ ช่วยให้สามารถเข้าถึง services ที่รันบนพอร์ต 80 ภายในคอนเทนเนอร์ผ่านพอร์ต 8080 บนเครื่องโฮสต์

\- \*\*image nginx:\*\* ระบุชื่อ image ที่ต้องการรัน ซึ่งในตัวอย่างนี้ใช้ image "nginx"

เมื่อมีการสั่ง docker run จะมีตรวจสอบว่ามี image นั้นอยู่ในเครื่องหรือไม่ ถ้าหากไม่พบ Docker ก็จะทำการ Pull Image นั้นๆจาก Docker Hub ลงมาให้

การใส่ Option ต่างๆสำหรับแต่ละ Image จะไม่เหมือนกัน มีการ map port หรือ mount volume ต่างกัน โดยเราสามารถดูได้จากคำอธิบายของแต่ละ Image ได้ในเว็บ \[hub.docker.com]\(https\://hub.docker.com/)

Port ที่เรากำหนดด้วย -p สามารถเข้าถึงได้จาก IP Address ของเครื่อง (localhost, 127.0.o.1 หรือ IP ในวงแลน) ตามด้วย Port ที่เรากำหนดไว้ ในที่นี้คือ \[http\://localhost:8080/]\(http\://localhost:8080/) (ถ้าใช้ Docker Toolbox จะเป็น IP : 192.168.99.100 โดย default)

โดย Container ที่เราลองรันอยู่เป็น Web Server เพราะฉะนั้นจะต้องมาลองเปิดดูใน browser พบว่าสามารถใช้งานได้แล้ว

```dockerfile
docker ps -a
```

\- \*\*หยุด Container

\*\*เมื่อต้องการให้ container หยุดทำงานจะต้องใช้คำสั่ง docker stop ตามด้วย Container ID หรือ Name ที่ใช้อ้างอิง Container

```dockerfile

docker stop a715

```

ในที่นี้จะใช้ Container ID ในการอ้างอิงเพื่อสั่งหยุด contianer ที่ต้องการ

(การเรียกใช้ Container ID ไม่จำเป็นต้องใส่ให้ครบทุกตัวอักษรก็ได้)

\- \*\*ลบ Container

\*\*เมื่อไม่ใช้งาน Container นั้นแล้ว ไม่อยากจะใช้ต่อสามารถลบ Container ได้ด้วยคำสั่ง

```dockerfile

docker rm a715

```

จะใช้คำสั่งนี้ได้ก็ต่อเมื่อ Container นั้นหยุดทำงานไปแล้ว

\*\*Docker Image ทั้งหมด\*\*

```dockerfile
docker images
```

การแสดงผลหลังจากใช้งานคำสั่ง docker images

\*\*คำสั่งลบ Image

\*\*เมื่อต้องการลบ image จะต้องแน่ใจก่อนว่ามี container ไหนเรียกใช้ image นี้อยู่รึเปล่าถ้าไม่มีการเรียกใช้งานก็สามารถสั่งลบ image ด้วยคำสั่ง docker rmi ตามด้วย Image หรือ Image name ที่อ้างอิงกับ Image นั้นๆ ที่ต้องการ

\`\`\`

```dockerfile
docker rmi c07e
```

หรือ

```dockerfile
docker rmi nginx:stable-alpine
```

สำหรับคำสั่ง Docker เบื้องต้นที่ใช้กันอยู่บ่อยๆก็มีประมาณนี้ยังเหลือคำสั่งอีกมากมาย ซึ่งอาจจะพูดถึงในตอนต่อไปๆ ขอให้สนุกกับการใช้งาน Docker ครับ
